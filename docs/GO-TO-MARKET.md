## § 1 사전등록 → 초대 전환 SOP

### 사전등록 풀

- **출처**: Tally 폼 (`https://tally.so/r/japxg4`, 랜딩 페이지 히어로 CTA "알림받기")
- **누적**: 2026-06-30 기준 N명 (대기 중)
- **수집 항목**: 이메일, 선택 국가/지역, VTuber/더빙/연기 관심 선택

### 초대 흐름 (Phase 4 배포 후)

1. **초대 이메일 배치** (배포 D+1)
   - 템플릿: 「초대장을 받으셨습니다」+ 고유 초대 코드 + 직접 입장 링크
   - 일본 시간대 일괄 발송 (UTC+9)
   - **우선순위**: 일본/한국 + VTuber 관심 표시자 → 기타 (순차 T+24h 간격)

2. **초대 코드 활성화**
   - DB 테이블: `waitlist_invites` (email, code, invited_at, redeemed_at, redeemed_user_id)
   - RLS: 초대 코드 소유자만 리드/업데이트 (owner_verified_at 필드 함께 추가)

3. **초대 수용 확인**
   - 초대 이메일 오픈율 추적 (Resend 또는 SendGrid 메트릭)
   - 초대 코드 사용률 목표: **첫 주 20% 이상** (실측 후 조정)

## § 5 채널 자동화 실행 방식 (홍보 자동화 사례 조사, 2026-07-01)

> `/Users/family/jason/{mungmungfit, TaillogToss, vibehub-media}`의 실제 운영 중인 홍보 자동화 3건을 직접 대조 조사한 결과. 세 프로젝트 다 "API 코드 작성"보다 "메타 앱 심사·토큰 발급(사람만 가능)"과 "실제 cron 트리거 셋업"에서 막혔다 — ChatterBox는 이 삽질을 반복하지 않는다.

### 경로 비교: Manus 경유 vs Meta API 직접

| 경로 | 사례 | 방식 | 리스크 |
|---|---|---|---|
| **Manus 경유** | mungmungfit | Google Drive 폴더 폴링(1-발행대기→2-처리중→3-발행완료) 또는 Supabase 큐에 콘텐츠를 넣으면 Manus가 자체 "Instagram MCP"로 발행 | Manus도 Claude와 동일하게 자기 샌드박스에서 로컬 명령 실패 시 사람에게 재요청함 — "메타 소유 특권 API 접근"은 검증되지 않은 블랙박스 의존 |
| **Meta Graph API 직접** | TaillogToss, vibehub-media | 자체 Edge Function/백엔드가 Threads/Instagram Graph API 직접 호출 | 롱라이브드 토큰 **60일마다 수동 갱신 필요**(Meta 설계상 완전 자동화 불가) |

**채택**: ChatterBox는 Meta API 직접 경로를 기본으로 한다 (TaillogToss가 "L4: Manus AI 미도입"을 잠금 결정으로 명시한 것과 동일한 판단). Manus급 외부 에이전트는 로드베어링으로 쓰지 않는다.

### 채널별 실현성 (vibehub-media 실측 리서치 재사용)

| 채널 | 실현성 | 핵심 리스크 |
|---|---|---|
| **Threads** | 9/10 | 공식·무료·250건/일, App Review 부담 없음 — **1순위** |
| **YouTube Data API** | 9/10 | public 자동 업로드는 OAuth 앱 검증에 **2~6개월** 소요 → private 자동 업로드 후 운영자 1클릭 공개 전환으로 우회 |
| **Instagram Reels** | 6/10 | App Review + 공개 비디오 URL 필요 — vibehub-media는 결국 **폐기**, TaillogToss는 캐러셀만 유지 |
| X/Twitter, LinkedIn | — | 비용/오버헤드 대비 효과 낮아 vibehub-media가 **폐기** |

### ChatterBox 실행 순서

1. **1순위 — Threads 직접 API**: 개발 블로그/진행상황 요약 자동 발행 (TaillogToss `seed-threads-from-blog` 패턴 재사용 가능). Meta Developer App 등록 + Instagram Business 계정 연결은 사람만 가능한 단계이므로 Phase 0 코딩과 **병행 착수**.
2. **2순위 — Instagram은 당분간 수동**: Canva 캐러셀 수동 발행(TaillogToss M3.1 패턴). VGEN 클립 공유가 실제 트래픽을 만들기 시작하면 자동화 재검토.
3. **YouTube Shorts**: ChatterBox는 VGEN이 이미 9:16 클립을 제품 기능으로 생성하므로, vibehub-media처럼 TTS·Remotion 파이프라인을 새로 만들 필요 없음 — 사용자 생성 클립(동의 획득 후)을 그대로 활용. private 업로드 후 수동 공개 전환.
4. **자동화 프롬프트 작성 ≠ 동작**: 자동화 스펙만 쓰고 실제 cron 트리거(Vercel Cron 등)를 안 붙이면 TaillogToss B5와 같은 "코드 완료·통합 대기" 함정에 빠진다. 트리거 셋업을 완료 기준에 명시적으로 포함할 것.

## § 6 성공 지표 (L3)

> 실행 후 2주차 검증 체크포인트

- 초대 수용률 (Redeemed / Sent) ≥ 20%
- 초대 이후 최초 방 생성까지 평균 시간 ≤ 1주
- 채널별 유입 분포 파악 (YouTube > Discord > X 예상, 실측 조정)
- DAU (사전등록자 중) 첫 주 10% 이상
