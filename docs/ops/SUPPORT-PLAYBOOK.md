## FAQ 대응 스크립트

### Q1. 방에 입장이 안 돼요

**먼저 확인할 것:**
1. 크롬 최신 버전인지? (Safari iOS는 WebRTC 제한 있음)
2. 카메라/마이크 권한이 허용되어 있는지?
3. 방 인원이 6명 가득 찼는지?

**스크립트:**
```
안녕하세요! 입장 오류가 발생했군요.
아래를 확인해 주실 수 있을까요?

1. 브라우저: Chrome 최신 버전을 권장합니다
2. 마이크 권한: 주소창 왼쪽 자물쇠 아이콘 → 마이크 허용
3. 방 인원 초과: 현재 방이 가득 찼을 수 있어요

그래도 안 된다면 스크린샷과 함께 #버그리포트 채널에 올려주세요!
```

---

### Q2. 아바타가 안 움직여요

**먼저 확인할 것:**
1. 웹캠이 연결되어 있는지?
2. 카메라 권한이 허용되어 있는지?
3. iOS Safari인지? (키보드 폴백 안내 필요)

**스크립트 (PC 사용자):**
```
웹캠 트래킹이 안 되는 경우, 아래를 확인해 주세요!

1. 설정 페이지 → 웹캠 목록에서 사용 중인 카메라 선택
2. 카메라 권한 재허용 (주소창 → 마이크/카메라 설정)
3. 다른 앱이 카메라를 점유 중이면 해당 앱 종료 후 시도
```

**스크립트 (iOS 사용자):**
```
iPhone/iPad에서는 아바타 자동 트래킹이 지원되지 않아요.
대신 키보드로 표정을 선택할 수 있습니다!

1~5 키를 눌러 표정을 변경해 보세요:
[1] 기쁨 [2] 놀람 [3] 슬픔 [4] 화남 [5] 기본
```

---

### Q3. 소리가 안 들려요

**스크립트:**
```
소리 문제가 발생했군요! 아래 순서로 확인해 주세요.

1. 설정 페이지 → 오디오 출력 장치 선택 확인
2. 방에서 마이크 버튼이 음소거 상태인지 확인 (빨간색 = 음소거)
3. 스페이스바를 누르고 있을 때만 마이크가 켜지는 Push-to-Talk 모드인지 확인
4. 다른 참가자의 슬라이더가 0으로 내려가 있는지 확인

그래도 안 들리면 방을 나갔다가 다시 입장해 주세요.
```

---

### Q4. 크레딧이 차감됐는데 영상이 안 나와요

**스크립트:**
```
영상 생성 중 오류가 발생한 것 같습니다. 정말 죄송해요!

확인 절차:
1. VGEN 패널에서 "내 생성 목록" 탭 확인 → 상태가 "실패"로 표시되면 크레딧 환불 처리
2. 일시적 오류면 1~2분 후 다시 시도해 주세요

크레딧 환불이 필요하면 Tally 폼으로 문의해 주세요.
[가입 이메일 / 생성 시도 시각 / 스크린샷]을 함께 보내주시면 빠르게 처리할게요!
```

---

### Q5. 방장 권한이 사라졌어요

**스크립트:**
```
방장 권한이 분실되는 경우가 있어요. 아래를 시도해 주세요.

1. 방을 나간 후 재입장 (방장 권한 자동 재부여 - 방 소유자 기준)
2. 다른 방장이 권한을 위임받은 경우, 해당 참가자가 방장 패널에서 권한 반납 가능

해결이 안 되면 방 ID와 함께 #버그리포트에 남겨주세요!
```

이메일로 문의해 주시면 수동으로 처리해 드릴게요.

## 버그 리포트 접수 프로세스

### 1. 접수 확인 (48시간 이내)

Discord `#버그리포트` 채널에서 확인 후 이모지 반응으로 접수 표시:
- 👀 = 확인 중
- ✅ = 재현 완료, 이슈 등록됨
- ❓ = 추가 정보 필요

**추가 정보 요청 템플릿:**
```
감사합니다! 재현을 위해 아래 정보가 필요합니다.
- 브라우저 및 버전 (예: Chrome 126)
- 운영체제 (Windows 11, macOS 15, iOS 17...)
- 오류 발생 시각 (가능하면)
- 오류 재현 단계 (어떤 순서로 클릭했는지)
- 스크린샷 또는 화면 녹화
```

### 2. 재현 및 심각도 판정

| 기준 | 우선순위 |
|------|---------|
| 서비스 사용 불가 / 데이터 손실 | P0 — 즉시 처리 (INCIDENT-PLAYBOOK.md) |
| 핵심 기능 오류 (방 입장·VGEN·크레딧) | P1 — 다음 배포 전 처리 |
| 부분 기능 오류 (UI 깨짐·특정 환경) | P2 — 스프린트 내 처리 |

## 크레딧 환불 처리

**조건:** fal.ai 에러로 생성 실패했으나 크레딧이 차감된 경우.

**처리 절차:**
1. Supabase Dashboard → `vgen_jobs` 테이블에서 해당 사용자·시각의 잡 조회
2. 잡 상태가 `failed`이고 `credit_deducted_at IS NOT NULL`이며 `credit_refunded_at IS NULL`이면 환불 대상
3. `refund-credit` Edge Function을 호출한다. 수동 `profiles`/`credits` UPDATE 금지.
```bash
supabase functions invoke refund-credit --body '{"user_id":"[user_id]","amount":[amount],"reason":"support_refund","ref_id":"[job_id]","idempotency_key":"support-[ticket_id]"}'
```
4. `credit_transactions(reason='refund')`와 `vgen_jobs.credit_refunded_at` 기록을 확인한다.
4. 사용자에게 Discord/이메일로 환불 완료 안내

## 홍보 이벤트 시 CS 모드

주요 홍보(미디어 카버리지, 인플루언서 공유, 커뮤니티 바이럴 등)로 트래픽이 급증했을 때 고객 지원 대응을 확대하는 절차.

### 급증 감지 기준

| 지표 | 기준값 | 조치 |
|------|--------|------|
| 일일 신규 가입 | 어제 대비 3배 이상 | CS 알림 모드 활성화 |
| Discord 인입 | 시간당 10건 이상 | 온콜 체계 전환 |
| #버그리포트 미해결 | 누적 5건 이상 | 에스컬레이션 채널 개설 |

### 감지 시 조치

1. **우선순위 재분류** (AGENT-OPS.md "CS 인입 분류 매트릭스" 준용)
   - P0_보안: 즉시 처리 (기존 SLA 유지)
   - P0_서비스불가: 일일 신규 3배 이상 시에만 48→24시간으로 단축
   - 버그_일반: 임시 자동응답 안내 후 배치 처리 (주 3회)

2. **자동응답 안내문 예시**
   ```
   안녕하세요! ChatterBox를 이용해 주셔서 감사합니다.
   현재 많은 문의가 들어오고 있어 응답이 지연될 수 있습니다.
   
   • 급할 경우: [FAQ 링크] → Q1~Q6 확인
   • 결제/계정: Tally 폼 제출 (우선 처리)
   • 버그 리포트: #버그리포트 채널에 스크린샷 첨부
   
   감사합니다! 곧 회신하겠습니다.
   ```

3. **에스컬레이션 채널**
   - 긴급 보안(P0_보안): Slack `#security-alerts` (즉시)
   - 서비스 불가(P0_서비스불가): Slack `#cs-critical` + 주인님 직접 알림
   - 버그 블로킹(버그_블로킹): Slack `#dev-log` + 개발팀 배정

### 정상화 복귀

- 일일 신규 가입이 전주 평균 이하로 내려갈 때까지 유지
- 미해결 누적 1건 이하가 되면 해제
