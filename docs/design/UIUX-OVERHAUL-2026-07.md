## 0. BLUF

ChatterBox를 치지직(탐색)·힉스필드(생성)·Cluster(입장)의 문법으로 재구성한다. 핵심 전환은 세 가지: ① 홈을 마케팅 히어로가 아니라 **"지금 입장할 수 있는 것들의 그리드"**로, ② 아바타·쇼츠·무대 생성을 **글로벌 [+ 만들기] 1클릭 진입**으로, ③ **구경은 비로그인, 참여만 로그인**으로. 색·radius 토큰은 그대로 두고, 제품 화면용 타이포·모션 티어만 추가한다.

## 2. 공통 문법 — 제품 UI 6원칙

1. **미디어가 주인공, 크롬은 저채도.** 5곳 모두 UI 채도 0.21–0.46. 썸네일·아바타가 색을 내고 버튼·네비는 뉴트럴/브랜드 1색. 우리 적용: 크림/딥그린 크롬 + `accent #56F09F`는 **LIVE 배지와 주 CTA에만**(치지직의 민트 절제 패턴).
2. **제품 타이포는 dense.** 작동하는 3곳(치지직·cluster·vroid)의 h1은 20–28px, 본문 12–16px. 마케팅 display 스케일(46–88px)은 방 밖 제품 화면에서 금지.
3. **flat + 그림자 최소.** 그림자는 오버레이·드롭다운·썸네일 호버에만. 카드 구분은 헤어라인 보더(우리 `border #E6E2D2`)로.
4. **모션 150–300ms ease-out.** 랜딩의 500ms flecto 이징은 마케팅 전용. 제품 화면은 hover 150 / 전환 200 / 오버레이 300.
5. **홈 = 지금 입장할 수 있는 것들.** 히어로 대신 라이브 그리드(치지직) 또는 개최 중 목록(cluster)이 첫 화면.
6. **구경 무료, 참여 로그인.** 시청·둘러보기는 비로그인, 채팅·출연·생성 시점에 로그인 요구(치지직 게이트).

## 3. IA 재설계

**현재:** `/` → HomeRedirect → 로그인 강제 → `/lobby/{theater,workshop,teahouse,atelier}` 전부 ProtectedRoute. 방문자는 문 앞에서 회원가입부터 요구받는다.

**제안:**

```
/                    디스커버리 홈 (비로그인 공개)
├─ LIVE 지금 열린 무대     ← 공개방 목록, 카드 그리드 [치지직]
├─ 다시보기               ← 공연 녹화 갤러리
├─ 아바타 쇼케이스          ← 공개 설정된 아바타 카드 [vroid hub]
└─ [+ 만들기] 글로벌 버튼   ← 헤더 상시 노출 [힉스필드]
    ├─ 아바타 만들기 → 의상실 (3스텝 위저드: 업로드→자동리깅→보정)
    ├─ 쇼츠 만들기   → 제작소 (프리셋 카드 갤러리에서 시작)
    └─ 무대 열기     → 대극장 방 생성

/live /replays /avatars   공개 딥링크 라우트 (SEO·공유용)
/rooms/:id                공개 프리뷰(제목·씬·인원) → [관전 입장(게스트)] / [출연(로그인)]
좌측 글로벌 네비: 홈 · 대극장 · 제작소 · 찻집 · 의상실 · 내 작품
```

로그인 게이트 이동: 페이지 진입(현재) → **행위 시점**(방 만들기·출연·채팅·생성 실행).

## 5. 단계별 개발 방향

> **실행 현황 (2026-07-16, phase-loop 완주)** — 상세는 GAP-MATRIX 진행 로그 동일 날짜 4건.
> - **P0 관전 개방 ✅** 익명 게스트 관전(getAppUser 기본거부+화이트리스트) + `list-public-rooms`(Public·레이트리밋)
> - **P1 디스커버리 홈 ✅** `/` = 공개 광장 + LiveRail + 앱 타입 스케일(§8)
> - **P2 만들기 허브 ✅** 글로벌 [+ 만들기] 3진입(전부 기존 플로우 딥링크). 쇼츠 프리셋 갤러리는 defer(VGEN 템플릿 시스템 필요)
> - **P3 포털 연동 ✅(부분)** 랜딩(snack-web-submit) Halls에 LiveNow 스트립(list-public-rooms 소비, 실패 시 무해 폴백). **`/replays`·`/avatars` 공개 갤러리는 defer** — 녹화·아바타를 "공개로 전환"하는 발행 UI가 아직 없어 갤러리가 공허해짐. 선행 조건: visibility 발행 플로우(후속 트랙).
> - **배포 대기(승인)**: Supabase Anonymous sign-in 활성화 · Edge deploy 4종 · CF Pages · 랜딩 재배포 · 라이브 2탭 실증.

| 페이즈 | 내용 | 핵심 변경 | 난이도 중심 |
|---|---|---|---|
| **P0 관전 개방** | 공개방 목록 비로그인 열람 + 게스트 관전(viewer) 입장 | ProtectedRoute 게이트를 행위 시점으로 이동, 게스트 세션 정책(Supabase anonymous auth 검토) | 인증·보안 |
| **P1 디스커버리 홈** | `/`를 HomeRedirect → 디스커버리 홈으로 교체 | LIVE 카드 그리드·다시보기 레일·[+ 만들기] 헤더, app 타이포/모션 티어 신설 | UI 신규 |
| **P2 만들기 허브** | 아바타 3스텝 위저드 + 쇼츠 프리셋 갤러리 진입 | 기존 의상실·제작소 기능을 위저드/갤러리 셸로 감싸기(기능 재사용, 진입만 재설계) | UX 플로우 |
| **P3 공개 갤러리·포털** | `/replays` `/avatars` 공개 페이지 + 랜딩 repo(snack-web-submit)가 같은 공개 API를 읽어 SEO 포털화 | 공개 read API(엣지 함수, Supabase 직결 금지) | API·SEO |

각 페이즈 게이트: 실렌더 스크린샷 검증 + 비로그인 시나리오 워크스루("링크만 받은 심사위원이 30초 안에 무대를 구경할 수 있는가").
