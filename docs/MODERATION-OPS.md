# 모더레이션 운영 워크플로

> G-147 산출 문서. 콘텐츠 신고 처리·이의제기·운영자 판단 기준.
> G-47 Admin Review Console 계약서 및 G-121 CommunityGuidelines.md 연계.

---

## 운영자 역할 및 권한

| 역할 | 권한 | Supabase 역할 |
|------|------|-------------|
| **Super Admin** | 모든 콘텐츠 관리, 계정 영구 정지, 이의제기 최종 판단 | `admin` |
| **Moderator** | 신고 검토, 임시 정지 (7일), 경고 발송 | `moderator` |

## 신고 큐 처리 절차

### 1. 큐 진입

```
사용자 신고 제출 → moderation_reports 테이블 INSERT (status: 'pending')
→ Admin Review Console 신고 큐에 자동 노출
```

### 2. 일일 검토 주기

**매일 오전 10시 KST** Admin Console(`/admin/reports`) 접속:

```
1. 신고 큐를 "접수 시각 오래된 순"으로 정렬 (기본값)
2. 각 신고를 열어 아래 판단 프로세스 진행
3. 처리 후 상태를 `resolved_action` / `resolved_noaction` / `dismissed`로 업데이트
```

### 3. 판단 프로세스

```
신고 내용 확인
  ↓
CommunityGuidelines.md의 금지 항목과 비교
  ↓
┌────────────────────────────────┐
│ 명백한 위반?                    │
│ (폭력·성인 컨텐츠·혐오 발언)     │
└────────────────────────────────┘
     Yes ↓              No ↓
   즉시 조치            맥락 확인
   (아래 §4)           (최근 3건 신고 이력 확인)
                            ↓
                    이전 경고 있음? → 경고 발송 또는 임시 정지
                    처음? → 경고 또는 DISMISS
```

### 4. 조치 종류 및 기준

| 조치 | 기준 | 실행 |
|------|------|------|
| **DISMISS** (기각) | 규정 위반 아님, 오해 또는 분쟁 | 신고 상태 `dismissed` 업데이트 |
| **경고 발송** | 첫 경미한 위반 (욕설·저급 표현) | 인앱 알림 + 이메일 발송 |
| **임시 정지 7일** | 두 번째 위반 또는 중간 위반 | `moderation-action` Edge Function → `account_restrictions` INSERT |
| **임시 정지 30일** | 세 번째 위반 또는 중간 위반 반복 | `moderation-action` Edge Function → `account_restrictions` INSERT |
| **영구 정지** | 심각한 위반 (불법 콘텐츠·사기) | `moderation-action` Edge Function → `account_restrictions` INSERT (`expires_at=NULL`) |
| **콘텐츠 삭제** | 특정 게시물만 규정 위반 | 해당 레코드 삭제/soft-delete |

## 이의제기 처리 (G-92 연계)

### SLA: 72시간 이내 응답

이의제기 제출 → `vgen_appeals` 테이블 INSERT (G-104 연계)

**처리 절차:**

```
1. 이의제기 검토 큐 확인 (매일 오전 10시)
2. 원 신고 내용 + 이의제기 내용 함께 검토
3. 판정 기준:
   - 새로운 증거가 있는가?
   - 당시 판단이 잘못됐는가?
   - 규정이 불명확했는가?
4. 72시간 이내 결정 → 인앱 알림 발송
```

**이의제기 응답 템플릿:**

```
안녕하세요, [닉네임]님.

이의제기 검토 결과를 안내드립니다.

[원 결정 유지인 경우]
검토 결과, [위반 사유]에 해당하는 것으로 판단됩니다.
원 결정을 유지합니다. 궁금한 점은 이메일로 문의해 주세요.

[이의제기 인용인 경우]
검토 결과, [이의제기 사유]가 타당하다고 판단했습니다.
조치를 해제하고 계정을 복구했습니다. 불편을 드려 죄송합니다.
```

## 신고 처리 결과 통보 템플릿

신고 처리가 완료되면 신고자와 피신고자에게 다음 규칙에 따라 통보합니다:

**신고자(신고 제출자)에게 통보:**
```
안녕하세요, [신고자 닉네임]님.

귀하가 신고한 콘텐츠/사용자에 대한 검토가 완료되었습니다.

신고 ID: [신고_ID]
신고 일자: [신고_날짜]
처리 결과: [승인/기각/부분 처리]
처리 내용: [구체적 조치 설명, 예: "콘텐츠 삭제 완료" 또는 "규정 위반 미확인으로 기각"]

이의가 있으시면 이메일로 문의해 주세요.
```

**피신고자(신고받은 사용자)에게 통보:**
```
안녕하세요, [피신고자 닉네임]님.

귀하의 콘텐츠/행동에 대한 신고가 접수되어 검토했습니다.

처리 결과: [승인/기각/부분 처리]
처리 내용: [구체적 조치 설명, 예: "콘텐츠 삭제 완료", "계정 경고", "임시 정지" 등]
이의 신청: 72시간 내에 gmdqn2tp@gmail.com으로 이의를 신청할 수 있습니다.

이 통보에는 신고자(신고를 제출한 사용자)의 정보(이름, 이메일, 사용자ID)가 포함되지 않습니다.
신고자의 신원은 보호됩니다.
```

**중요: 신고자 정보 보호**
- ❌ 신고자의 사용자명, 이메일, ID를 피신고자에게 절대 노출
- ❌ 신고자의 개인정보를 신고 처리 결과 통보에 포함
- ✅ 신고자의 신원을 기밀 유지한다고 명시적으로 피신고자에게 알림
- ✅ 신고자는 신고 ID를 통해 진행 상황 추적 가능

## 녹화 동의 위반 신고 처리

녹화 동의를 받지 않고 영상을 녹화 또는 공개한 경우의 처리 기준:

| 위반 유형 | 신고 카테고리 | 첫 위반 조치 | 재위반 조치 | 즉시 영구 정지 |
|---------|-----------|----------|-----------|------------|
| **동의 없이 녹화 후 저장** | 프라이버시 침해 | 즉시 녹화 soft-delete + 경고 | 임시 7일 정지 | - |
| **동의 없이 녹화 영상 공개** | 프라이버시 침해 + 미디어 오용 | 즉시 영상 비공개(private) 처리 + 경고 | 임시 30일 정지 | - |
| **동의 철회 후 계속 노출** | 지속적 침해 | 즉시 영상 비공개 + 경고 | 임시 14일 정지 | - |

### 처리 절차

**1단계: 신고 접수 및 검증**
```
- 신고자가 동의 위반 사실 명시 (예: "녹화 동의 거절했으나 영상 공개됨")
- 방 ID, 녹화 일시, 관련 사용자 확인
- 해당 recordings.consent_json에서 신고자의 동의 상태 확인
  * consented: false → 동의하지 않은 것 확인
  * consent_type: 확인 필요 (pre/post)
```

**2단계: 조치**
- **동의 없는 녹화**: recordings 행을 soft-delete 또는 상태를 'discarded'로 변경
- **동의 있으나 공개**: recordings.visibility를 'private'으로 변경, 호스트에게 재공개 전 모든 참가자 재동의 요구
- **호스트 누적 경고**: 같은 호스트의 3회 위반 시 계정 정지(§4 "조치 종류 및 기준" 참조)

**3단계: 피해자 통보**
```
안녕하세요, [피해자 닉네임]님.

귀하가 신고한 녹화 동의 위반에 대해 다음과 같이 처리되었습니다:

방 ID: [room_id]
녹화 일시: [녹화_날짜]
처리 내용: [영상 삭제/비공개 처리 등]
호스트 조치: [경고/임시 정지 등]

향후 해당 호스트와의 방에서는 더욱 주의를 기울여 주시고,
계속된 위반 시 즉시 신고해 주세요.
```

## 관리자 SQL 쿼리

### 신고 현황 조회

```sql
-- 처리 대기 신고 목록
SELECT
  r.id,
  r.created_at,
  r.category,
  r.description,
  r.status,
  r.reporter_user_id,
  p.display_name AS reported_user
FROM moderation_reports r
LEFT JOIN users p ON p.id = r.reported_user_id
WHERE r.status = 'pending'
ORDER BY r.created_at ASC;
```

### 사용자 정지 처리

```sql
-- 실제 제재는 API-SURFACE.md `moderation-action` Edge Function으로 실행하고
-- account_restrictions에 canonical restriction을 남긴다.
-- 아래 SQL은 운영 큐 상태 확인/수동 복구용이다.
UPDATE moderation_reports
SET status = 'resolved_action',
    handled_at = now(),
    handled_by = '[admin_user_id]'
WHERE id = '[report_id]';
```

### 주간 신고 통계

```sql
SELECT
  DATE_TRUNC('week', created_at) AS week,
  COUNT(*) AS total_reports,
  COUNT(*) FILTER (WHERE status IN ('resolved_action', 'resolved_noaction')) AS reviewed,
  COUNT(*) FILTER (WHERE status = 'dismissed') AS dismissed,
  COUNT(*) FILTER (WHERE status = 'pending') AS pending,
  COUNT(*) FILTER (WHERE status = 'resolved_action') AS actions
FROM moderation_reports
WHERE created_at >= now() - interval '8 weeks'
GROUP BY week
ORDER BY week DESC;
```

## Japan Policy Review Gate (G-173)

일본 출시는 번역만으로 처리하지 않는다. 아래 항목은 법률 자문이 아니라, 출시 전 법무/운영 검토 체크리스트다.

### 출시 전 검토 항목

- [ ] 일본어 커뮤니티 가이드라인: 금지 콘텐츠, 신고 사유, 제재 단계의 일본어 원문 작성.
- [x] 동인/二次創作 가이드: 원작 캐릭터·상표·음원·대본 사용 시 허용/삭제 기준을 별도 문서화. → `docs/legal/FANART-POLICY.md` 작성 완료
- [ ] 성인/청소년 보호 기준: 연령 게이트, 공개 데모룸 접근 제한, 18세 이상 콘텐츠 처리 정책.
- [ ] 일본 권리자 문의 흐름: 삭제 요청 접수, 본인 확인, 임시 비공개, 이의제기 SLA.
- [ ] 현지 운영자 리뷰: 일본어 뉘앙스, 은어, kaomoji/밈 맥락을 반영한 신고 판단.

### 일본 출시 전 MUST NOT

- 글로벌 Moderation API 결과만으로 일본 런칭 승인하지 않는다.
- 二次創作를 "모두 허용" 또는 "모두 금지"로 단순화하지 않는다.
- 법률/저작권 문의를 Discord 공개 채널에서 처리하지 않는다.

### 인플루언서 및 협업 크리에이터 홍보 고지 (G-257)

협찬, 파트너십, AI 활용 콘텐츠의 명확한 고지 의무:

#### A. 협찬 표시 의무
- **협찬 콘텐츠**: 브랜드/서비스 무료 지원, 금전 보상, 상품 증정을 받은 콘텐츠
  - 필수 표기: `[협찬]`, `[AD]`, `[Sponsored]` 제목 또는 설명에 명시
  - 목적: 시청자가 광고 콘텐츠임을 즉시 인지
- **처리**: 협찬 표기 누락 시 운영 공지 후 7일 내 수정 요청. 불응 시 콘텐츠 비공개 가능

#### B. AI 활용 명시
- **AI 생성 콘텐츠**: [TERMS-OF-SERVICE.md § 5.2.1](TERMS-OF-SERVICE.md#521-ai-생성-콘텐츠-공개-시-표시-의무) 참조
  - `[AI생성]` 태그 강력 권장 (public/members_only 배포 시)
- **AI 더빙/보이스변환**: AI 기술 사용 여부 설명에 명시
  - 목적: 청취자가 자연 음성인지 AI 음성인지 구분 가능하게 함

#### C. 협업/수익 공유 투명성
- **공동 크리에이터 명시**: 더빙, 편집, 사운드 디자인 등 협력자 이름 명시
- **수익 공유 콘텐츠**: "이 영상의 수익은 협력자와 공유됩니다" 명시 권장
- **독점 배포 계약**: YouTube 독점 배포 등 특정 플랫폼 계약 존재 시 명시

#### D. 신고 및 검증
- 협찬 표기 누락 신고: [MODERATION-OPS.md](MODERATION-OPS.md) 커뮤니티 신고 플로우 이용
- 운영자 검증: 협찬사와 크리에이터 간 기록(이메일, 계약서) 요청 가능
- 허위 협찬 신고: 악의적 신고로 판단 시 신고자 페널티 적용

## User Safety Controls (G-175)

운영자용 큐만으로는 사용자가 안전하다고 느끼기 어렵다. 주요 표면에는 직접 누를 수 있는 신고·차단·이의제기 플로우가 있어야 한다.

| UI | 위치 | 최소 동작 |
|---|---|---|
| 신고 버튼 | 참가자 메뉴, 채팅 메시지, 생성물 상세 | 사유 선택, 설명, 증거 첨부, 접수 번호 반환 |
| 차단 버튼 | 참가자 프로필/메뉴 | 즉시 같은 방 입장 차단, 차단 목록에 추가 |
| 차단 목록 | Settings > Safety | 차단 해제, 차단 일시, 사유 메모 표시 |
| 계정 제재 appeal | 로그인 차단 화면/이메일 링크 | 제재 사유 확인, 500자 설명, 72시간 SLA |
| 녹화 동의 | 녹화 시작 전 모달 + 방 상단 배지 | 동의/거절, 거절 시 녹화 제외 또는 viewer 전환 |

신고/차단/appeal 이벤트는 `analytics_events`에도 남기되, 신고 본문과 증거 파일은 PII 보호 정책에 따라 별도 보관한다.

## Data Transparency UI (G-176)

개인정보처리방침을 읽지 않은 사용자도 현재 데이터 흐름을 이해할 수 있어야 한다.

| 상황 | UI 표시 |
|---|---|
| GreenRoom 카메라/마이크 테스트 | "현재 녹화 중 아님" 배지, 로컬 처리/전송 여부 표시 |
| LiveKit 방 입장 | 음성/영상이 방 참가자에게 전송됨을 표시 |
| VGEN 생성 전 | "프롬프트와 선택한 미디어가 fal.ai로 전송됩니다" 확인 |
| 녹화 시작 | 방 상단 `REC` 배지, 참여자별 동의 상태 |
| 데이터 저장 위치 안내 | Settings > Privacy에 LiveKit/Supabase/R2/fal.ai 역할 요약 |

투명성 UI는 법률 문서의 축약본이다. 버튼을 눌렀을 때 전문 문서로 이동할 수 있어야 하지만, 전문 문서 클릭을 동의의 대체 수단으로 삼지 않는다.
