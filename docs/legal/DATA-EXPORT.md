# snack-web 데이터 반출 및 계정 삭제 정책

**시행일**: 2026-06-30
**최종 수정일**: 2026-06-30

## 1. 개요

snack-web(이하 "서비스")는 이용자의 데이터 이동 및 삭제 권리를 존중합니다.

본 정책은 다음을 규정합니다:

- **데이터 반출**: 서비스에서 개인 데이터를 내보내기 (export)
- **계정 삭제**: 계정을 영구 삭제하고 데이터를 처리하는 절차
- **30일 유예**: 삭제 요청 후 즉시 비활성화하고 30일 뒤 영구 삭제하는 절차 안내

본 정책은 「개인정보보호법」 제35조(정보주체의 권리), GDPR Article 20(데이터 이동권)을 기반으로 합니다.

## 2. 데이터 반출 (Data Export)

### 2.1 반출 대상 데이터

다음 데이터를 구조화된 형식으로 반출할 수 있습니다:

#### A. 계정 정보
```json
{
  "account": {
    "email": "user@example.com",
    "display_name": "username",
    "timezone": "Asia/Seoul",
    "preferred_genres": ["drama", "comedy"],
    "created_at": "2026-01-15T10:30:00Z",
    "last_login": "2026-06-28T15:45:00Z"
  }
}
```

#### B. 콘텐츠 목록

| 콘텐츠 유형 | 포함 정보 | 포맷 |
|-----------|----------|------|
| **업로드 영상** | ID, 제목, 설명, 업로드일시, URL, 공개 범위 | JSON + 파일 |
| **AI 생성 영상** | 생성 프롬프트, 생성일시, 결과 URL, fal.ai 생성ID | JSON + 파일 |
| **더빙** | 더빙 대상 영상ID, 음성 파일, 생성일시 | JSON + 파일 |
| **녹화** | 방장 여부, 참여자 목록, 녹화 시간, 파일 | JSON + 파일 |
| **채팅** | 채팅방ID, 메시지 텍스트, 작성 시간 | JSON/CSV |

#### C. 거래 및 활동 기록

| 항목 | 포함 정보 | 포맷 |
|------|----------|------|
| **크레딧 거래** | 거래ID, 유형(구매/사용), 금액, 날짜 | JSON/CSV |
| **참여 기록** | 참여한 공연ID, 참여 날짜, 역할(실연자/관객) | JSON/CSV |
| **신고/차단** | 신고한 사용자, 차단 목록, 이유 (선택) | JSON |

### 2.2 반출 불가 데이터

다음 데이터는 반출 불가:

- **비밀번호 해시**: 보안 상 저장 형태로 제공 불가
- **세션 토큰**: 보안 상 비공개
- **감사 로그(Audit Logs)**: 운영자용 보안 기록
- **제3자 소유 데이터**: 다른 사용자의 개인정보 (예: 더빙 참여자 연락처)
  - 단, 자신과의 상호작용 기록(채팅, 반응)은 포함
- **서버 로그**: 접속 IP, User-Agent (보안상 제한)

### 2.3 반출 요청 방법

**이메일로 요청:**

```
받는 사람: gmdqn2tp@gmail.com
제목: [데이터 반출 요청] username (또는 이메일)
```

**요청 본문 (필수):**

```
안녕하세요,

[데이터 반출 요청]

아래 정보를 확인하고 데이터 반출을 요청합니다.

계정 정보:
- 이메일: [가입 이메일]
- 사용자명: [snack-web username 또는 display_name]
- 가입일: [대략적 날짜]

반출 원하는 데이터 유형 (중복 선택 가능):
□ 계정 정보
□ 모든 콘텐츠 (영상, 더빙, AI 생성 등)
□ 채팅 기록
□ 거래 및 활동 기록
□ 전체 데이터

반출 포맷 선택 (복수 선택 가능):
□ JSON
□ CSV (텍스트 데이터만)
□ MP4 (영상 파일)
□ ZIP (모두 묶음)

연락처: [회신 이메일 또는 전화]

나는 위 요청이 사실이며, 본인의 데이터임을 확인합니다.

[서명 또는 "동의합니다" 명시]
```

### 2.4 본인 확인 절차

#### 1단계: 이메일 접수
- 요청 받은 이메일로부터 신원 확인

#### 2단계: 확인 메일 발송
- 가입 시 등록한 이메일로 확인 링크 발송
- 확인 링크 유효 기간: 48시간

#### 3단계: 확인 완료
- 링크 클릭 또는 별도 인증 (2FA 있는 경우)
- 미인증 시: 데이터 반출 취소
- 앱 내 요청은 `data-export-request` Edge Function만 사용한다.
- 요청 전 최근 재인증(비밀번호 재입력 또는 OAuth reauth)을 요구한다.
- 서버는 export manifest를 생성해 본인 소유/권한 범위만 포함하고 `audit_logs(event_type='data_export_requested')`를 남긴다.

#### 보안 예외
- 계정에 로그인된 상태에서 요청한 경우: 본인 확인 생략 가능
- 법적 증명(신분증 사본 등) 요청 가능 (필요시)

### 2.5 처리 기한 및 타임라인

| 단계 | 기한 | 설명 |
|------|------|------|
| **접수 및 확인** | 1~2 영업일 | 이메일 접수 및 본인 확인 요청 |
| **본인 확인** | 2일 (이메일 유효 기간) | 사용자의 확인 링크 클릭 |
| **데이터 수집** | 7 영업일 | 데이터베이스에서 데이터 추출 |
| **파일 생성** | 3 영업일 | JSON, CSV, MP4 등 포맷 변환 |
| **서명 URL 생성** | 2 영업일 | Cloudflare R2 다운로드 링크 생성 |
| **전달** | 1 영업일 | 사용자에게 다운로드 링크 이메일 발송 |

**총 처리 기한: 15 영업일 이내** (GDPR, PIPL 준수)

### 2.7 다운로드 링크 및 보안

#### 링크 특성
- **서명 URL**: Cloudflare R2 보안 서명 URL
- **유효 기간**: 24시간
- **다운로드 횟수 제한**: 1회
- **재발급**: 만료/사용 후 새 요청 필요. 재발급도 재인증과 감사 로그 필수

#### 보안 주의
- 링크는 본인 이메일에만 발송
- 링크를 공유하면 제3자가 데이터 접근 가능
- ZIP 생성과 링크 발급은 브라우저가 아니라 서버 작업 큐에서 수행한다.
- 링크 유효 기간 후 자동 삭제

### 2.8 대용량 파일 처리

#### 임계값
- **전체 데이터 > 2GB**: 분할 다운로드 제공
  - Part 1, Part 2, ... 등으로 나뉨
  - 각 파트는 별도 다운로드 링크

#### 예시
```
- export_part1_videos.zip (1.5GB, 유효기간 7일)
- export_part2_metadata.zip (0.8GB, 유효기간 7일)
```

### 2.9 빠른 반출 (긴급)

#### 일반 반출과의 차이
- 기한: **영업일 3일 이내**
- 포맷 제한: JSON + 중요 파일만
- 비용: 추가 비용 없음

#### 긴급 반출 신청
- 이메일 제목: **[긴급 데이터 반출 요청]**
- 이유 설명 필수 (예: "계정 해킹 우려")

## 3. 계정 삭제 (Account Deletion)

### 3.1 삭제 가능 여부 확인

#### 삭제 가능
- 모든 일반 이용자
- 법적 제약 없는 상태

#### 삭제 불가 (일시적)
- 진행 중인 저작권 신고 피고인 상태
- 미해결 거래(환불 대기 중)
- 계정 정지 상태

#### 삭제 불가 (영구)
- 불가능 (모든 계정은 삭제 권한 있음)

### 3.2 계정 삭제 요청 방법

**이메일로 요청:**

```
받는 사람: gmdqn2tp@gmail.com
제목: [계정 삭제 요청] username (또는 이메일)
```

**요청 본문:**

```
안녕하세요,

[계정 삭제 요청]

아래 계정을 삭제해주세요.

계정 정보:
- 이메일: [가입 이메일]
- 사용자명: [username]

삭제 전 확인:
□ 데이터 반출이 필요하면 먼저 신청했습니다.
□ 계정은 즉시 비활성화되고 30일 후 영구 삭제되며, 30일 후에는 복구할 수 없음을 이해합니다.
□ 공유/협작 중인 콘텐츠의 처리 방식을 확인했습니다 ([UGC-OWNERSHIP.md] 참조).

나는 위 요청이 사실이며, 계정 삭제를 최종 확인합니다.

[서명 또는 "동의합니다" 명시]
```

### 3.3 삭제 전 체크리스트

삭제를 요청하기 전, 다음을 확인하세요:

- [ ] **중요 데이터 백업?** 데이터 반출 완료했나요?
- [ ] **구독 취소?** 월정액이 있으면 먼저 취소하세요 (환불 불가).
- [ ] **공유 콘텐츠?** 자신의 더빙 또는 녹화 영상 처리 방식 확인.
- [ ] **미해결 거래?** Stripe 거래가 진행 중이 아닌지 확인.
- [ ] **차단/신고 상태?** 계정 정지 상태가 아닌지 확인.

### 3.4 본인 확인 및 최종 확인

#### 1단계: 이메일 접수
- 요청받은 이메일로부터 신원 확인

#### 2단계: 최종 확인 이메일
```
제목: [계정 삭제 최종 확인] alice_dancer

안녕하세요,

귀하의 계정 삭제 요청을 받았습니다.

계정: alice_dancer (user@example.com)
요청 날짜: 2026-06-30

[주의]
계정 삭제를 확인하면 즉시 비활성화됩니다:
- 로그인과 공개 노출 중단
- 30일 후 영구 삭제
- 30일 후 계정명 회수 불가
- 콘텐츠 일부 삭제 불가 (협작 콘텐츠·법적/evidence hold)

최종 확인: 아래 링크를 클릭하여 삭제를 진행하세요.
[확인 링크] (유효기간 48시간)

또는 "동의합니다"를 회신하세요.
```

#### 3단계: 최종 확인 완료
- 링크 클릭 또는 회신 확인
- 미확인 시: 요청 자동 취소 (48시간 후)

### 3.5 삭제 처리 기한

| 단계 | 기한 |
|------|------|
| 접수 | 1 영업일 |
| 최종 확인 메일 발송 | 1 영업일 |
| 최종 확인 기간 | 48시간 |
| 삭제 실행 | 1 영업일 |
| **총 기한** | **최대 7 영업일** |

### 3.6 삭제 후 데이터 처리

#### 30일 유예 후 삭제 대상

다음 데이터는 계정 삭제 확인 즉시 비활성화되고, 30일 유예 기간 후 완전히 제거됩니다:

- 개인정보 (이메일, 비밀번호 해시, 타임존, 선호 장르)
- 계정 프로필 정보
- 세션 토큰 및 인증 정보
- 접속 로그 (IP, User-Agent)
- 채팅 메시지
- 더빙 오디오 파일
- 개인 프롬프트 (AI 생성)

#### 보존 대상

다음 데이터는 제한된 기간 보존됩니다:

| 데이터 | 보존 이유 | 기한 | 처리 방식 |
|--------|----------|------|----------|
| **업로드 영상** | 타 사용자의 더빙/협작 권리 보호 | 협작 없으면 삭제, 협작 있으면 90일까지 보존 | 계정명만 "[삭제된 사용자]"로 익명화, 더빙 제공자의 음성/이름 유지 |
| **더빙 참여 음성** | 더빙 제공자의 재산권 | 합성 결과물이 존재하는 동안 | dub_tracks.user_id는 제거, speaker_name과 음성 파일은 유지 |
| **녹화 영상** | 공동 소유권 | 다른 실연자 있으면 90일까지, 없으면 즉시 삭제 | 계정명 익명화, 요청 시 자신의 얼굴만 흐림 처리 |
| **거래 기록** | 세법상 의무 | 5년 (소비자기본법) | 개인식별 가능 정보만 마스킹 |
| **감사 로그** | 보안 기록 | 90일 | 전체 삭제 |

#### 익명화 처리

다음 데이터는 **익명화 후 보존**:

- 시청 통계 (개인식별 불가)
- 참여 기록 (user ID만 남음)
- 더빙 참여 목록 (이름 제거, ID만 남음)

### 3.7 계정 삭제 후 복구

#### 30일 유예 중 복구
- **복구 기간**: 최종 확인 후 30일 동안 계정은 비활성화 상태로 보관됩니다.
- **복구 방법**: 동일 계정으로 로그인 후 복구 요청 또는 등록 이메일 확인 링크로 취소합니다.
- **30일 후**: 영구 삭제되어 복구할 수 없습니다.
- **예외**: 법적 의무·결제·보안·evidence hold 데이터는 계정 복구와 별개로 접근제한 보존될 수 있습니다.

### 3.8 계정명 회수

#### 삭제 후 계정명 사용
- **즉시 사용 불가**: 삭제 후 최소 90일 후 새 사용자가 사용 가능
- **이유**: 초대 링크, 멘션 등의 orphan 참조 방지

#### 같은 이메일로 재가입
- **가능 여부**: 삭제 후 90일 경과 후 가능
