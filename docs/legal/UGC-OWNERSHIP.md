# snack-web UGC(사용자 생성 콘텐츠) 소유권 정책

**시행일**: 2026-06-30
**최종 수정일**: 2026-06-30

## 1. 기본 원칙

snack-web(이하 "서비스")에서 사용자가 생성, 업로드, 제작한 모든 콘텐츠(이하 "UGC")는 **원칙적으로 사용자가 소유**합니다.

서비스 운영자는 서비스 운영 목적으로만 제한된 라이선스를 취득합니다.

본 정책은 다양한 콘텐츠 유형별로 소유권 범위를 명확히 합니다.

## 2. UGC 정의

### 2.1 포함되는 콘텐츠

| 콘텐츠 유형 | 설명 | 예시 |
|-----------|------|------|
| **동영상** | 사용자가 업로드한 원본 영상 파일 | mp4, webm 등 |
| **음성/더빙** | 사용자가 녹음하거나 제공한 음성 | wav, mp3 |
| **텍스트** | 채팅, 댓글, 소개 | 사용자 메시지 |
| **프롬프트** | AI 영상 생성 시 입력한 텍스트 | "a woman dancing in..." |
| **라이브 스트림** | 웹캠을 통한 실시간 영상 | 방송 중 영상 |
| **메타데이터** | 제목, 설명, 태그 | "내 첫 공연", "#dance" |

### 2.2 제외되는 콘텐츠

다음은 사용자 소유가 아닙니다:

- 서비스 제공자가 제공한 배경, 프레임, UI 요소
- 제3자 저작권 콘텐츠 (저작권자 동의 없이 사용된 음악, 영상)
- AI 기반 모델의 기여도 (fal.ai의 Seedance 알고리즘 자체)

## 3. 콘텐츠 유형별 소유권

### 3.1 원본 영상(Upload)

```
소유권자: 업로드 사용자 (100%)
서비스 운영자 라이선스: 서비스 내 표시, 처리, 배포, 공유 권리만
제3자 라이선스: 없음 (업로드자 책임)
```

**세부 사항:**

- **소유 권리**: 영상 파일의 저작권은 완전히 사용자에게 있습니다.
- **서비스 라이선스 범위**:
  - 서비스 플랫폼에서 재생 가능하도록 처리 (인코딩, 스트리밍 최적화)
  - 설정된 공개 범위에 따라 다른 이용자에게 표시 및 배포
  - 썸네일, 미리보기 생성
  - 서비스 분석 목적의 집계 통계 (시청 수, 좋아요 등)
  - **AI 학습 데이터로 사용 금지** ← 명시적 거부

- **공개 범위 설정**: 사용자는 업로드 시 공개 범위를 선택합니다.
  - `public`: 모든 이용자가 볼 수 있음
  - `members_only`: 서비스 가입자만 볼 수 있음
  - `private`: 업로드자와 초대 사용자만 볼 수 있음

### 3.2 실시간 스트림(웹캠)

```
소유권자: 스트림 송출자 (100%)
저장 여부: 녹화 동의 없이는 저장 금지
서비스 라이선스: 실시간 처리 및 표시만 (저장 금지)
```

**세부 사항:**

- **실시간 처리**: 웹캠 영상은 실시간으로만 처리됩니다.
- **저장 금지**: 명시적 동의 없이 녹화되지 않습니다.
- **자동 삭제**: 실시간 방송 종료 시 스트림 데이터는 즉시 폐기됩니다.
- **녹화 동의**: 방장(또는 모든 참여자)이 "녹화 시작" 버튼을 누르면, 모든 참여자에게 알림이 표시되고 동의 팝업이 나타납니다.

### 3.3 녹화 영상(Recording)

```
소유권자: 방장 및 실연자 공동 소유
저장 기한: 90일
삭제 권한: 방장이 전체 삭제 가능, 개별 실연자는 자신의 얼굴 흐림 처리 요청 가능
공유 권한: 방장이 결정 (모든 공동 소유자의 동의 필요 권장, 단 명확한 계약 없을 시 방장 판단)
```

**세부 사항:**

- **공동 소유 범위**: 방장 + 실연자들 공동 저작물
- **보존 기간**: 90일간 Cloudflare R2에 저장
- **자동 삭제**: 90일 경과 후 자동 삭제
- **방장의 삭제 권한**: 방장은 녹화 영상을 언제든 완전히 삭제할 수 있습니다 (되돌릴 수 없음).
- **개별 실연자 권리**:
  - 자신의 얼굴만 흐림 처리(blur) 요청 가능
  - 녹화 영상 사용 방지 요청 가능 (새로운 용도로의 공유/배포)
  - 요청 처리: 5 영업일 내
- **공유 및 배포**:
  - 방장은 `public`, `members_only`, `private` 중 선택 가능
  - 타 플랫폼으로의 다운로드/재공유 시 모든 공동 소유자의 동의 필요 (권장)

### 3.4 더빙(Dubbing)

```
더빙의 구성 요소:
├─ 원본 영상(업로드 영상): 업로드자 소유
├─ 더빙 음성: 더빙 제공자 소유
└─ 합성 결과물: 저작권 공동 (업로드자 + 더빙 제공자)
```

**세부 사항:**

#### A. 더빙 음성 (더빙 제공자 소유)
- 더빙 중 녹음한 음성의 소유권은 **더빙 제공자(음성 제공자)**에게 있습니다.
- 더빙 제공자는 자신의 음성 사용을 언제든 취소할 수 있습니다.
- 음성 취소 시, 합성 결과물에서 해당 더빙은 제거됩니다.

#### B. 합성 결과물 (공동 저작물)
- 더빙이 적용된 영상(원본 영상 + 더빙 음성)은 **업로드자와 더빙 제공자의 공동 저작물**입니다.
- 원본 영상 업로드자는 더빙 결과물의 최종 배포 여부를 결정합니다.
- 더빙 제공자는 원본 업로드자의 동의 없이 합성 결과물을 배포할 수 없습니다.

#### C. 보존 기간
- 더빙 오디오: 90일
- 합성 결과물: 원본 영상 삭제 시까지 또는 90일 (정책 검토 필요)

#### D. 사용 및 배포
- 더빙이 적용된 영상의 공개 범위는 원본 영상 업로드자가 결정합니다.
- 더빙 제공자는 다음을 할 수 없습니다:
  - 합성 결과물을 자신의 계정에서 재공유
  - 합성 결과물을 타 플랫폼에 재배포
  - 자신의 음성을 원본 영상과 분리하여 판매/재사용

### 3.5 AI 생성 영상(Seedance)

```
소유권자: AI 영상 생성자(프롬프트 입력자) - 단, fal.ai의 AI 모델 라이선스 제약
라이선스: fal.ai 이용약관 우선 적용
생성 프롬프트: 생성자 소유 (fal.ai로 전송됨)
생성 결과물: 생성자 소유 (fal.ai 라이선스 하에서)
```

**세부 사항:**

- **생성 소유권**: AI로 생성한 영상의 소유권은 프롬프트를 입력한 사용자에게 있습니다.
- **fal.ai 라이선스 제약**:
  - fal.ai의 Seedance 모델은 특정 라이선스(예: CC-BY-NC 등)로 배포될 수 있습니다.
  - 상업적 사용 제약이 있을 수 있습니다.
  - 생성자는 fal.ai의 이용약관을 확인할 책임이 있습니다.
  
  **확인 URL**: https://fal.ai/privacy, https://fal.ai/terms

- **프롬프트 전송**:
  - 프롬프트(입력 텍스트)는 fal.ai로 전송됩니다.
  - fal.ai의 개인정보처리방침이 적용됩니다.
  - 생성자는 민감한 정보를 프롬프트에 포함하면 안 됩니다.

- **생성 결과물 보존**:
  - 생성된 영상은 snack-web 서버에 저장됩니다.
  - 보존 기간: 90일 기본, 사용자가 삭제하면 즉시 삭제됩니다.
  - fal.ai에도 일시적으로 저장될 수 있습니다.

- **상업적 사용 제약**:
  - 생성 영상의 상업적 사용은 fal.ai의 라이선스 정책에 따릅니다.
  - snack-web에서 크레딧을 사용하여 생성한 영상도 마찬가지입니다.
  - 명확하지 않은 경우, fal.ai에 문의하세요: https://fal.ai

### 3.6 블렌드셰이프 얼굴 트래킹 데이터

```
소유권: 데이터 생성자(실연자) 소유
저장 여부: 저장 금지 (실시간만)
처리: 실시간 얼굴 표정 인식만
```

**세부 사항:**

- **실시간 처리만**: 웹캠에서 감지한 blendshape 데이터는 AI 영상 생성 목적으로 **실시간 처리만** 합니다.
- **저장 금지**: 블렌드셰이프 데이터는 저장되지 않습니다.
- **목적**: 사용자의 얼굴 표정을 AI 생성 영상의 캐릭터에 적용하기 위한 용도만
- **제3자 전송 금지**: fal.ai 등 제3자에게 전송되지 않습니다.

### 3.7 채팅 메시지

```
소유권: 메시지 작성자 소유
저장 기간: 채팅방 삭제 또는 계정 삭제 시 제거
공개 범위: 채팅방에 접근 가능한 사용자에게만 표시
삭제 권한: 작성자 또는 방장이 삭제 가능
```

**세부 사항:**

- **메시지 소유권**: 각 메시지는 작성자가 소유합니다.
- **삭제**: 작성자 또는 채팅방 관리자(방장)가 메시지를 삭제할 수 있습니다.
- **편집**: 작성자는 메시지 편집 가능 (최대 24시간 내)
- **보존**: 채팅방이 존재하는 동안만 보관

## 4. 서비스 운영자의 라이선스 범위

### 4.1 명시적 권리

서비스 운영자는 다음 목적으로만 UGC를 사용할 수 있습니다:

- **서비스 내 처리**: 재생, 스트리밍, 인코딩, 최적화
- **서비스 내 표시**: 프로필, 피드, 검색 결과에 표시
- **콘텐츠 관리**: 저작권 침해, 불법 콘텐츠 감시 (자동화 도구 포함)
- **기술 개선**: 서비스 성능 분석, 오류 추적
- **통계 집계**: 익명화된 시청 수, 참여도 등 (개인식별 불가)
- **법적 의무**: 법원 명령, 수사 기관 요청 (영장 필요)

### 4.2 명시적 거부

서비스 운영자는 다음을 **절대 할 수 없습니다**:

- **AI 학습**: UGC를 AI 모델 학습 데이터로 사용
- **마케팅 목적 외부 배포**: 서비스 외부에서 UGC를 광고/홍보용으로 사용
- **상업적 재판매**: UGC를 제3자에게 판매 또는 라이선스
- **변형 및 2차 저작물**: UGC를 원본 의도와 다르게 변형 (단, 더빙 등 협업 콘텐츠는 제외)
- **신원 도용**: UGC의 저작권을 운영자 명의로 주장

### 4.3 제3자로의 라이선스 전달

서비스 운영자가 취득한 라이선스는 제3자에게 양도할 수 없습니다.

**예외:**
- 서비스 인수/합병 시: 새 운영자는 동일 범위의 라이선스만 승계
- 사용자 명시적 동의: (예: "YouTube에 클립 공유" 기능)

## 5. 공개 범위 및 접근 권한

### 5.1 공개 범위 설정

사용자는 업로드 시 다음 중 선택:

| 범위 | 접근 권한 | 검색 노출 | 공유 가능 |
|------|----------|----------|----------|
| **public** | 모든 이용자 | O | O |
| **members_only** | 가입 사용자만 | 제한적 | O (로그인 필요) |
| **private** | 본인 + 초대자 | X | 초대 링크만 |

### 5.2 검색 및 발견(Discovery)

- `public` 콘텐츠만 검색 결과에 노출
- `members_only`는 가입자 피드에만 표시
- `private`은 검색 불가

### 5.3 다운로드 권한

- **기본**: 다운로드 금지 (스트리밍만)
- **사용자 설정**: 사용자가 "다운로드 허용" 옵션을 활성화하면 가능
- **다운로드 관리**: 사용자는 언제든 다운로드 권한 철회 가능

## 6. 계정 삭제 시 데이터 처리

### 6.1 즉시 삭제 대상

계정 삭제 즉시 제거:

- 개인정보(이메일, 비밀번호 해시, 개인 프로필)
- 채팅 메시지
- 더빙 오디오
- 웹캠 스트림 녹화(자신의 목소리/영상)

### 6.2 보존 대상 (공동 소유 또는 타 사용자 기여)

**삭제되지 않음 (공동 저작물 또는 타 사용자 기여):**

#### A. 업로드한 원본 영상
- **다른 사용자의 더빙이 있을 경우**: 완전 삭제 금지 (더빙 제공자의 권리 보호)
  - 처리: 계정명을 "[삭제된 사용자]"로 변경
  - 영상 자체는 보존되고, 더빙 기여자의 이름만 유지됨
  - 더빙 기여자가 음성 제거를 요청하면 그때 업로드자 동의 없이 제거 가능
- **타 사용자의 더빙이 없는 경우**: 즉시 삭제

#### B. 녹화 영상 (방장이 아닌 경우)
- **다른 실연자와 함께 녹화된 경우**: 보존 (공동 저작물)
  - 처리: 계정명 익명화 + "90일까지 보존" 명시
  - 요청 시 자신의 얼굴만 흐림(blur) 처리 가능 (음성/얼굴 제거는 방장/호스트만 결정)
  - 다른 실연자가 있으면 그들의 권리까지 영향받으므로 개인 요청으로는 삭제 불가
- **자신 혼자 녹화된 경우**: 즉시 삭제 가능

#### C. 더빙 참여 콘텐츠
- **더빙 제공자로서의 음성**: 삭제하지 않음
  - 처리: dub_tracks 테이블에서 user_id만 제거, speaker_name은 유지 (원본 영상과의 연결성 보존)
  - 음성 파일은 유지 (합성 결과물의 무결성)
  - 필요시 "더빙 제공자 제거" 요청 가능 → 그 트랙의 음성 제거 (업로드자 동의하에)
- **음성 사용 취소**: 더빙 제공자가 언제든 자신의 음성 사용을 취소할 수 있음 (G-112: 더빙 철회)
