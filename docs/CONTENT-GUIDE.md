# 텍스트 수정 가이드 (팀원용)

사이트에 보이는 **모든 글자는 딱 한 파일**에서 관리합니다:

```
src/content/content.ts
```

컴포넌트(`.tsx`)는 건드릴 필요가 없습니다. 이 파일의 값만 바꾸면 사이트 전체에 반영됩니다.

## 1. 가장 자주 하는 일

### 브랜드명 정하기
파일 맨 위 `BRAND` 객체에서:
```ts
export const BRAND = {
  company: "SNACK",       // 회사/브랜드명
  name: "ChatterBox",     // 내부 플랫폼명
  url: "https://example.com",
  applyUrl: "...",        // 사전등록/얼리엑세스 폼 링크
  contactEmail: "hello@example.com",
};
```
`BRAND.company`는 SNACK, `BRAND.name`은 ChatterBox입니다. 헤더·푸터·메타·비교표는 이 값을 가져갑니다.

### 카피 교체
지금 들어 있는 `〔仮〕…` 는 전부 **임시 텍스트**입니다. 확정 문구로 바꾸세요.
예) 히어로 제목:
```ts
hero: {
  title: "画像1枚から、\n動くVTuberへ。",  // \n = 줄바꿈
  subtitle: "...",
}
```

## 2. 구조 (위에서부터 화면 순서)

| 키 | 화면 섹션 |
|---|---|
| `meta` | 브라우저 탭 제목 · OG(공유 카드) |
| `nav` | 상단 메뉴 |
| `hero` | 첫 화면 (제목·버튼·지표) |
| `logos` | 신뢰 로고 띠 |
| `problem` | 문제 제기 카드 3개 |
| `how` | 사용법 3스텝 |
| `features` | 기능 그리드 (아이콘 6개) |
| `comparison` | 비교 표 |
| `useCases` | 활용 사례 |
| `pricing` | 요금 플랜 3개 |
| `faq` | 자주 묻는 질문 |
| `finalCta` | 마지막 행동 유도 |
| `footer` | 푸터 |

## 3. 규칙 / 주의

- **따옴표 안 텍스트만** 바꾸세요. 키 이름(`title:` 등)·콤마·중괄호는 그대로.
- 줄바꿈은 `\n`. 항목 추가는 배열에 `{ ... }` 를 콤마로 이어서.
- `features` 의 `icon` 값은 정해진 키만 가능: `zap` `smile` `video` `cpu` `globe` `sliders`.
  (다른 아이콘이 필요하면 개발자에게 `Features.tsx` 의 `ICONS` 매핑 추가 요청.)
- 링크 `href`: 같은 페이지 섹션은 `#features` 처럼 `#`+id. 외부는 전체 URL.

## 4. 카피 검증 규칙

카피를 작성하거나 수정하기 전에 아래 체크리스트를 따르세요.

### 기능 정확성 검증

- **기능 ID 확인**: `features` / `comparison` / `useCases` 의 각 항목을 `docs/FEATURE-SPEC.md`의 기능 ID와 대조하세요.
  - 예: "최대 6명이 동시에 연기" → `ROOM-02` (무대 레이아웃 엔진) 확인
  - 예: "1분 버튜버 생성" → `MOD-01~MOD-02` (모델 선택·캘리브레이션) + `VGEN` 생성 기능 확인
  
- **완성도 라벨 확인**: `docs/PITCH-READINESS.md`의 3단계 라벨을 따릅니다.
  - `Live now` = 배포되어 사용 가능한 기능. 현재형/완결형 표현 허용 ("지금 바로 할 수 있어요", "바로 나와요")
  - `PoC proven` = 로컬/스테이징에서 증명됨. 조건부 현재형 ("만들어줄 수 있어요", "설계 중")
  - `Designed` = 명세만 완성. 미래형 필수 ("할 수 있을 거예요", "곧 나올 예정")
  
- **금지 표현**: 아래 표현은 실제 구현 기능에만 사용하세요.
  - ❌ "바로" "즉시" "지금" "바로" → Live now만
  - ❌ "최대" "최고" (수치) → 기술 명세와 정확히 일치 확인 후
  - ❌ "AI가 자동으로" → 자동화 범위 (FEATURE-SPEC.md §기능 세부사항) 확인

### 지역별 컨텍스트

다국어 카피 작성 시 `docs/MODERATION-OPS.md`의 지역 정책 섹션과 연계하세요 (특히 일본).

## 5. 일본 시장 카피 검수 체크리스트

일본 시장 대상 카피(`ja.ts`, 메타 설명, 광고 텍스트)를 작성·수정할 때 아래 항목을 검토하세요.
[[MODERATION-OPS]] 의 Japan Policy Review Gate와 연동됩니다.

### 과장광고 규제 (경품표시법 · 약관표시법)

| 항목 | 확인 사항 |
|-----|---------|
| **기술적 사실 명시** | "AI가 자동으로" / "바로" 표현은 처리 시간 / 조건 / 한계를 명시했는가? 예: "1~3분" (정확한 범위) vs "빠르게" (모호함) |
| **비교 표 객관성** | 타사(CustomCast/Discord Stage 등)와의 비교는 검증 가능한 사실만 포함했는가? 주관적 표현("더 낫다", "최고") 금지 |
| **영상/이미지 괴리** | 데모 영상·스크린샷은 실제 제품과 일치하는가? 목업·렌더링은 "미리보기" / "イメージ" 라벨 필수 |
| **한계·조건 공시** | 기능의 제약(지원 브라우저·디바이스·네트워크 요구사항·동시 사용자 수 등)을 명시했는가? |
| **가격·크레딧** | 요금제 설명에서 환율·추가 비용·구독 자동 갱신 여부를 명시했는가? |

### 아동 보호 및 개인정보 표기

| 항목 | 확인 사항 |
|-----|---------|
| **연령 제약 명시** | 13세 미만 사용 불가 조건이 메타·FAQ·온보딩에 명시되어 있는가? (FEATURE-SPEC.md AUTH-06, ONBOARDING-03 연계) |
| **개인정보 수집 고지** | 웹캠·마이크·위치 권한 요청 전에 목적·저장소(Supabase/LiveKit)·보유 기간을 명시했는가? |
| **부모 동의 프로세스** | 13~18세 사용 시 부모 동의 요청 흐름이 명세된 페이지 링크를 포함했는가? |

### 콘텐츠 안전성

| 항목 | 확인 사항 |
|-----|---------|
| **생성 콘텐츠 고지** | AI 배경 생성(`ROOM-01` AI Scene) / AI 자막 추출(`Video Dub`) 기능이 있다면, 인앱에서 "AI 생성 콘텐츠" 마크가 표시되는가? |
| **모더레이션 정책 링크** | CommunityGuidelines (금지 콘텐츠·신고 절차) 링크가 FAQ 또는 하단에 명시되어 있는가? |
| **2차 창작 정책** | "버튜버 캐릭터로 기존 작품 더빙" 기능(영상 더빙 VGEN 기능) 사용 시, 저작권·초상권 주의 문구를 FAQ에 추가했는가? |

## 6. 다국어 (나중에)

지금은 일본어(`ja`)만 있습니다. 한국어/영어를 추가하려면 `ja` 블록을 통째로 복사해
`content` 객체의 `ko`/`en` 키를 채우면 됩니다(현재는 `ja`로 폴백 중).

## 5. 미리보기

```bash
npm install
npm run dev      # http://localhost:3000
```
저장하면 즉시 화면에 반영됩니다(HMR).
