# cs-triage

## 트리거

다음 패턴에서 자동 실행:
- "CS 분류", "cs triage", "고객 문의 처리", "인입 분류"
- AGENT-OPS.md § CS 인입 분류 루틴 호출 시

---

## 목적

Discord `#도움말` / `#버그리포트` + Gmail(Tally 폼) 인입을 읽어  
분류 → 응답 초안 작성 → Slack 보고까지 한 루틴에서 처리한다.

---

## 실행 절차

### Step 1 — 인입 수집

```
[Gmail]
Gmail MCP: search_threads(query="from:tally is:unread", max_results=20)
→ 제목·본문 요약

[Discord / Slack 미러]
Slack MCP: slack_read_channel(channel="#cs-inbox", limit=50)
→ 읽지 않은 메시지 추출
```

### Step 2 — 분류

각 인입에 대해 아래 기준으로 태그를 부여한다.

| 태그 | 키워드 / 패턴 | 우선순위 |
|---|---|---|
| `P0_보안` | "해킹", "데이터 유출", "내 계정이 도용", "취약점", "비밀번호 노출" | 즉시 |
| `버그_블로킹` | "입장이 안 됨", "앱이 안 열림", "로그인 오류", "방이 사라짐", "결제가 안 됨" | 높음 |
| `버그_일반` | "아바타가 안 움직임", "소리가 안 들림", "화면이 깨짐", "느림" | 보통 |
| `기능요청` | "~기능 추가해줘", "~하면 좋겠다", "이런 기능은 없나요?" | 낮음 |
| `문의_일반` | "어떻게 사용하나요", "~는 어디서 설정하나요", "가격이 얼마인가요" | 낮음 |
| `결제_계정` | "환불", "탈퇴", "계정 삭제", "구독 취소" | 높음 (주인님 전달) |

복합 태그 가능 (예: `버그_블로킹 + 결제_계정`).

### Step 3 — 대응 생성

#### P0_보안
```
1. Slack #security-alerts 즉시 전송:
   "🚨 [P0_보안 인입] {요약} — 즉시 조사 필요"
2. INCIDENT-PLAYBOOK.md 해당 섹션 열어 대응 절차 확인
3. Gmail로 사용자에게 "접수 확인 + 24시간 내 연락" 응답
4. 주인님 직접 알림 (Gmail MCP)
```

#### 버그_블로킹
```
1. SUPPORT-PLAYBOOK.md에서 FAQ 매칭 (Q1~Q5 체크)
   - 매칭됨: 해당 스크립트로 응답 초안 작성
   - 미매칭: 재현 시도 필요 항목으로 분류
2. 재현 시도:
   - 계약서 + SecurityPolicies.md에서 해당 기능 흐름 추적
   - 문서 기준에서 버그 원인 추정 가능 여부 판단
3. Slack #dev-log 보고:
   "[버그_블로킹] {요약} — 재현 시도 결과: {결과}"
4. 응답 초안 Discord/Gmail에 작성
```

#### 버그_일반 / 문의_일반
```
1. SUPPORT-PLAYBOOK.md FAQ Q1~QN 순서로 키워드 매칭
2. 매칭 스크립트로 응답 초안 작성 (한국어)
3. 초안을 Slack #cs-inbox에 붙여넣기:
   "✉️ 응답 초안 [{태그}] — 확인 후 발송 or 자동 발송"
```

#### 기능요청
```
1. GAP-MATRIX.md에서 이미 등록된 Feature ID인지 확인
2. 등록됨: "현재 {G-XX} 로드맵에 포함되어 있습니다" 응답 초안
3. 미등록: GAP-MATRIX에 후보 행 추가 (상태: LATER), 감사 응답 초안
4. Slack #dev-log: "[기능요청] {요약} — GAP-MATRIX {상태}"
```

#### 결제_계정
```
1. 자동 대응 금지 — 사람(주인님) 검토 필수
2. Gmail MCP로 주인님에게 포워딩:
   "[CS 에스컬레이션 - 결제/계정] {요약}"
3. 사용자에게 "72시간 내 담당자 연락" 확인 응답
```

### Step 4 — Slack 스윕 보고

```
[CS 스윕] {datetime}
━━━━━━━━━━━━━━━━━━━━
신규 인입: N건
  P0_보안: 0건
  버그_블로킹: N건
  버그_일반: N건
  기능요청: N건
  문의_일반: N건
  결제_계정: N건
━━━━━━━━━━━━━━━━━━━━
응답 초안 작성: N건
에스컬레이션: N건
미해결 누적: N건
```

→ Slack `#cs-inbox` 전송

Step 5 — AGENT-OPS.md 갱신

`docs/status/AGENT-OPS.md` 상단 frontmatter:
- `last_cs_sweep: YYYY-MM-DDTHH:MM` 갱신
- `open_issues` 수 갱신
- "현재 열린 이슈" 표에 미해결 항목 추가

---

## 응답 언어·톤

- 언어: 한국어 기본 (사용자가 영어로 쓰면 영어로 응답)
- 톤: 친근하고 명확. "~해주세요" 존댓말. 기술 용어는 최소화.
- 길이: 3~5문장. 해결이 복잡하면 단계별 번호 목록.
