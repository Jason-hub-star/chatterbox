# /backlog

도그푸딩 감사 백로그(`docs/DOGFOOD-AUDIT-2026-07.md §0`)를 **우선순위 순으로 하나씩 실제 구현**한다. 검증은 성역(추측 금지), 배포는 승인 게이트. `$ARGUMENTS` 로 범위 지정 가능:
- `/backlog` → **트랙 A**의 다음 미완 1건 (A-P0→A-P1→A-P2). **트랙 B(UIUX 표현)는 자동 선택 안 함**
- `/backlog A` → 트랙 A 미완 전부 (배포 지점·분기서 정지)
- `/backlog B` → 트랙 B(UIUX 1회 패스) — A의 seam 위에 스타일. A가 안정된 뒤 실행 권장
- `/backlog SEC-3` → 특정 항목
- `/backlog <다른 백로그 경로>` → 그 문서 §0 체크리스트를 소스로

## Steps (각 항목마다)

1. **타깃 선정** — `docs/DOGFOOD-AUDIT-2026-07.md §0` **트랙 A**에서 미완(`[ ]`) 최상위 항목을 고른다(기본은 트랙 A만 — 로직·보안·seam. 트랙 B(UIUX 표현)는 `$ARGUMENTS`에 `B`가 있을 때만). §1 표의 `정수정` + `file:line` + 연관 계약서/스펙을 읽는다.
2. **구현(최소)** — 전역 코드 미니멀리즘(YAGNI·재사용·표준→네이티브→최소코드). 근본원인을 고친다(호출처 grep). 안 시킨 추상화·의존성 금지. 프리미티브가 여러 항목을 닫으면 프리미티브 먼저(예: rate-limit 프리미티브 = SEC-1·SEC-4).
3. **검증(성역, 스킵·위조 금지)** — 변경 유형에 맞는 그라운드 트루스:
   - 프론트(ts/tsx): `npm run type-check` + `npm run lint` + `npm run test` (+ UI면 실렌더/비주얼 스모크)
   - Edge Function/마이그: `deno check --node-modules-dir=auto <fn>` + 격리 postgres 실측(`docker run --rm public.ecr.aws/supabase/postgres:17.6.1.127` 로 마이그+RPC 로직) 또는 `supabase-slice-verify` 스킬(통합/E2E)
   - 문서: `npm run docs:check`
   - **증거 없으면 `[x]` 금지.** 실행한 명령·결과만 근거로 쓴다.
4. **기록** — §0 체크박스 `[ ]→[x]` + 실측 요약 붙이고, `## 진행 로그` 에 한 줄(무엇·검증결과·배포여부). 문서만 바뀌면 `docs:check`.
5. **배포 게이트** — 라이브 반영에 배포/푸시/커밋이 필요하면 **실행하지 않는다**. "구현+실측 완료·배포 대기"로 표시하고 **다음 항목으로 계속**(배포는 모아서 승인받음).
6. **반복** — 범위 내 다음 항목으로. 배포 지점·진짜 분기(파괴적/모호/외부의존)만 정지.

## Guardrails

- **배포·push·commit 은 명시 승인 시에만.** `$ARGUMENTS` 에 `deploy` 가 있을 때만 배포 단계 수행.
- **검증은 절대 안 깎는다.** 실행 안 한 명령을 통과로 적지 않는다. 배포본이 소스보다 오래될 수 있음(소스 기준 명시).
- **외부런타임 의존 결과**(예: SEC-2 경로 traversal의 R2 정규화)는 라이브 테스트 없이 Confirmed 단정 금지 — 로직상 견고한 정수정(서버측 키 생성)을 적용하고 R2 실테스트는 배포 게이트로 넘긴다.
- **모델 분담**: 설계/리뷰는 메인이 직접(상위급이면). 넓은 탐색·독립작업만 서브에이전트(`Explore`/`general-purpose`), 감사 재실행은 `dogfood-audit` 스킬.
- 기능중심·UI 최소(사용자 선호): 백엔드/로직 제대로, 프론트는 최소 배선.
- 임시파일은 스크래치 디렉토리, 검증용 컨테이너는 `--rm`/실행 후 제거.

## 정지 시 보고 (BLUF)

- 이번 실행에서 `[x]` 된 항목 + **실측 증거**(명령·결과)
- **배포 대기** 목록(무엇을 배포하면 라이브되는지)
- 다음 미완 최상위 항목 + 정지 사유(배포 게이트/분기/범위완료)

## 종료

- 범위(기본=다음 1건, `P0`=P0 전부 등) 내 모든 항목이 구현+실측되면 요약하고 정지. 배포 대기분은 승인 요청으로 남긴다.
